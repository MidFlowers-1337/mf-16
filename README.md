# 博物馆展品外借管理系统

一个面向小型博物馆的本地 Web 应用，用于管理展品外借业务，包含展品信息维护、外借记录管理、风险检测提醒和运输安排功能。

## 技术栈

- **前端**：React 18 + TypeScript + Vite + Tailwind CSS + Zustand
- **后端**：Express.js 4 + TypeScript
- **数据库**：SQLite（better-sqlite3）
- **其他**：React Router DOM、Lucide React 图标

## 功能模块

1. **系统概览** - 统计数据、展品分布、近期运输
2. **展品管理** - 展品信息增删改查、分类/状态筛选
3. **借展机构** - 机构信息管理（含保险额度）
4. **外借记录** - 外借记录列表、标记归还、新建外借向导
5. **风险中心** - 时间冲突、恒温运输、保险额度、逾期等风险检测与处理
6. **运输安排** - 按日期时间线查看发出/归还排期

## 安装

### 环境要求

- Node.js >= 18
- npm（已内置在 Node.js 中）

### 安装依赖

```bash
cd d:\mf-16
npm install
```

## 启动

### 同时启动前后端（推荐）

```bash
npm run dev
```

启动后：
- 前端地址：http://localhost:5173
- 后端地址：http://localhost:3001

### 分别启动

```bash
# 仅启动前端（开发模式）
npm run client:dev

# 仅启动后端（开发模式，带热重载）
npm run server:dev
```

## 数据初始化

系统首次启动时会自动执行以下操作：

1. 在项目根目录创建 `museum.db` SQLite 数据库文件
2. 自动创建 5 张数据表：`exhibits`、`institutions`、`loan_records`、`loan_exhibits`、`risks`
3. 自动填充示例数据：
   - 10 件展品（青铜器、书画、陶瓷、玉器、织绣等类别）
   - 4 家借展机构（含保险额度信息）
   - 3 条外借记录（含 1 条逾期记录用于触发风险检测）

如需重置数据，删除 `museum.db` 文件后重启服务即可重新初始化。

## 功能验证指南

### 1. 系统概览页面

访问首页，应看到：
- 4 个统计卡片（展品总数、在借展品、活跃外借、未处理风险）
- 展品类别分布图
- 近期运输安排列表

### 2. 展品管理

- 点击左侧「展品管理」，应看到 10 条示例展品数据
- 点击「新增展品」按钮，填写信息后保存，列表应出现新记录
- 点击某展品的「编辑」按钮，修改信息后保存，列表应更新
- 使用搜索框、分类和状态筛选器，验证筛选功能

### 3. 借展机构

- 点击左侧「借展机构」，应看到 4 家示例机构
- 验证新增、编辑、删除机构功能
- 注意「保险额度」字段，这是风险检测的依据之一

### 4. 新建外借记录（核心功能）

1. 点击「外借记录」→「新建外借」
2. **第一步**：选择一家借展机构
3. **第二步**：勾选 1-2 件展品
4. **第三步**：填写借出/归还日期、运输方式、联系人
5. **第四步**：查看风险检测结果，验证以下场景：
   - 选择需要恒温的展品 + 普通运输 → 应出现中风险提醒
   - 选择估值高的展品 + 保险额度低的机构 → 应出现高风险提醒
   - 选择已在借的展品 + 日期重叠 → 应出现高风险提醒
6. 确认提交后，外借记录列表应出现新记录

### 5. 风险中心

- 点击左侧「风险中心」，应看到预生成的风险（含逾期风险）
- 点击严重程度筛选按钮（高/中/低），验证筛选
- 切换「显示已处理」开关，验证已处理风险的显隐
- 点击某风险的「标记已处理」按钮，风险状态应更新

### 6. 运输安排

- 点击左侧「运输安排」，应看到按日期分组的时间轴
- 「发出」用绿色标识，「归还」用蓝色标识
- 今天的日期用金色高亮
- 需恒温的展品旁显示温度计图标

### 7. 数据持久化验证

1. 在系统中新增一条展品或外借记录
2. 刷新浏览器页面（F5），数据应仍然存在
3. 停止服务（Ctrl+C），重新运行 `npm run dev`
4. 再次访问页面，之前新增的数据应依然存在
